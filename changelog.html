Product Update · Release Summary

CeleNation Fantasy

What changed in this release · Prepared for ownership review

This release moves CeleNation Fantasy from a demo prototype to a production-ready paid product. The application now stores real accounts in the cloud, accepts secure card payments, gives commissioners full control over their league, and includes a marketing demo visitors can explore without signing up.

Headline: Players can now sign up, pay the $35 season fee, and access their league from any device. Their roster, draft picks, and matchup results follow them everywhere — there is nothing for the commissioner to set up manually.

01. Real Accounts That Save Everywhere

Previously, the app stored everything inside the browser. If a player signed up on their laptop, they could not sign in from their phone — and clearing the browser would erase their roster. The app has been rebuilt with a cloud database, so every account, every roster, and every draft pick is stored centrally and follows the player to any device.

Before
Accounts lived only in the browser. Switching devices meant losing access. Clearing cookies erased everything.
Now
Sign in once, your league follows you. Phone, laptop, friend's computer — same league, same roster, instantly synced.

What this unlocks

  • Players can use the app from any phone, tablet, or computer.
  • Rosters, draft picks, weekly stats, and transactions are stored securely in the cloud.
  • Other players' draft picks appear automatically on everyone's screen — no manual refresh needed.

02. Stripe Paywall for the $35 Season Fee Revenue

The signup form now collects payment before creating an account. Players enter their info, click Continue to Payment ($35), complete the purchase on Stripe's secure checkout page, and are automatically signed in to a finished account.

How the payment flow works

  1. Player fills in username, password, security question, and avatar.
  2. Clicks Continue to Payment — redirected to Stripe's hosted checkout.
  3. Pays with any credit or debit card.
  4. Returned to the app, signed in, ready to play.

Safety nets built in

  • Closed-tab protection: if a player closes their browser mid-payment, a Stripe webhook still creates the account on the server side — no support tickets, no lost customers.
  • Duplicate-username protection: if two people pick the same name during checkout, only the first one is honored. The second gets a clear refund-eligibility message.
  • Test-mode first: the entire system was validated in Stripe test mode before being switched to live keys for real money.
Net effect: Every new account = $35 in revenue, collected automatically. Stripe handles the cards, the receipts, the refunds, and the disputes. The owner sees the money land in their bank within 2 business days of each charge.

05. Scoring Fix — Top 5 of 7 Players Bug Fix

The original code summed all 7 roster players when calculating weekly fantasy points. This was incorrect — only the 5 best performers each week should count toward a team's score. This has been corrected throughout the application.

Before
All 7 players' fantasy points were added together for the team total. There was no incentive to draft depth — every player counted equally.
Now
The 5 best performers each week count. The remaining 2 are bench depth, ready to step in if a star player has an off week. Visible on the My Team page with ✓ marks.

Where this applies

  • My Team page: roster now sorted by fantasy points; top 5 are marked with a green ✓, bottom 2 are dimmed.
  • Matchups: the week's score for both teams uses only top 5.
  • Standings: wins and losses now reflect proper top-5 scoring.
  • Championship bracket: playoff scoring is also top-5-only.

06. Championship Bracket — Top 4 From Each Group

The playoff format now matches the league rules:

  • Two groups of 8 players each play a 7-week round robin.
  • The top 4 from each group qualify for the championship — 8 qualifiers total.
  • These 8 players run a fresh re-draft from the full player pool (the "championship draft").
  • Standard 8-team single-elimination bracket: Quarterfinals → Semifinals → Final.

Seeding

The bracket uses cross-group seeding so the two #1 seeds can only meet in the Final, ensuring the best matchups happen at the right time:

09. Security & Reliability Security

Password security

  • Passwords are never stored in plain text. They are hashed with a per-account salt using scrypt — the industry standard for password storage.
  • Password hashes are not readable from the browser. Only the server-side functions (which have a separate secret key) can compare them. A leak of the public site's code would not expose any passwords.

Stripe integration security

  • Card data is handled entirely by Stripe. It never touches our servers.
  • Stripe webhook signatures are cryptographically verified, so no one can forge a fake "payment succeeded" event.
  • The Stripe secret key, Supabase service key, and admin bypass code are stored as environment variables — they are not in the public code.

Resilience

  • Two independent paths create the account after payment: the success page redirect AND the Stripe webhook. Whichever fires first wins. The other one safely no-ops.
  • A pre-existing latent bug in the scoring code (which would have crashed the app once any real draft completed) was caught and fixed before going live.
